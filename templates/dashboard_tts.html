FFmpeg로 전체 오디오 병합 4. 문장별 타임라인 계산 → SRT 자막 생성 ════════════════════════════════════════ ★ 음성 설정 (Voice Config) ════════════════════════════════════════ 【Google Cloud TTS (기본)】 - 언어: ko-KR (한국어) - 음성: ko-KR-Neural2-B (남성) / ko-KR-Neural2-C (여성) - 속도: 0.9x (자연스러운 속도) - 인코딩: MP3 【ElevenLabs (프리미엄)】 - 음성: Charon (역사 다큐 전용) - 안정성: 0.5 (자연스러운 변주) - 유사성: 0.75 (일관성 유지) - 스타일: 0.3 (차분한 나레이션) ════════════════════════════════════════ ★ 청킹 규칙 (Chunking) ════════════════════════════════════════ 【문장별 TTS 모드 (기본)】 - 문장 단위로 분할 (.!? 기준) - 최소 문자수: 10자 (너무 짧으면 병합) - 5000바이트 초과 시 추가 분할 【장점】 - 자막 싱크 정확도 높음 - 문장별 타이밍 제어 가능 - 끊김 없는 자연스러운 흐름 ════════════════════════════════════════ ★ SRT 자막 생성 ════════════════════════════════════════ 【타임라인 계산】 - 문장별 재생시간 균등 배분 - ffprobe로 실제 오디오 길이 측정 - 타임코드: HH:MM:SS,mmm 형식 【출력 예시】 1 00:00:00,000 --> 00:00:03,500 서기 391년이었어요. 2 00:00:03,500 --> 00:00:07,200 열여덟 살의 왕이 첫 출정에 나섰거든요. ════════════════════════════════════════ ★ 입력 형식 (Step3 Input) ════════════════════════════════════════ { "episode_id": "history_ep21", "language": "ko-KR", "voice": { "gender": "MALE", "name": "ko-KR-Neural2-B", "speaking_rate": 0.9 }, "scenes": [ { "id": "scene1", "narration": "..." }, { "id": "scene2", "narration": "..." } ], "sentence_mode": true } ════════════════════════════════════════ ★ 출력 형식 ════════════════════════════════════════ { "ok": true, "episode_id": "history_ep21", "audio_file": "outputs/audio/xxx_full.mp3", "srt_file": "outputs/subtitles/xxx.srt", "stats": { "total_duration_sec": 912.5, "total_chunks": 45, "successful_chunks": 45 } }
